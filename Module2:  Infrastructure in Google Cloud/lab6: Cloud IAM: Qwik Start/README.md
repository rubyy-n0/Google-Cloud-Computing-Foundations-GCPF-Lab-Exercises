# Cloud Run Functions: Qwik Start - Command Line 

## Objective
To learn how to create, deploy, and test a serverless Cloud Run Function using the gcloud command-line tool and Pub/Sub trigger.

## Key Steps Performed
1. Set the default Cloud Run region.
2. Created a Node.js function project directory.
3. Developed function in index.js using Functions Framework.
4. Created package.json file for dependencies.
5. Installed dependencies using npm install.
6. Deployed Cloud Run Function using gcloud CLI.
7. Configured Pub/Sub topic trigger (cf-demo).
8. Published test message using gcloud pubsub.
9. Verified function execution through Cloud Logging.
10. Read function logs using gcloud functions logs.

## Tools / Services Used
- Cloud Run Functions (Gen2)
- Cloud Shell
- gcloud CLI
- Pub/Sub
- Node.js
- Cloud Logging
- npm

## Results
Successfully deployed a serverless Cloud Run Function triggered by Pub/Sub messages and verified execution through logs.
