# Create a Virtual Machine (GSP001)

## Objective

To learn how to create and manage Compute Engine virtual machine instances using both the Google Cloud Console and the gcloud command-line tool.

## Key Steps Performed

1. Configured the project region and zone.
2. Created a VM instance named gcelab using the Cloud Console.
3. Configured machine type as e2-medium.
4. Enabled HTTP traffic through firewall settings.
5. Connected to the VM using SSH.
6. Updated the operating system packages.
7. Installed and verified the NGINX web server.
8. Accessed the NGINX welcome page through the VM's external IP address.
9. Created a second VM instance named gcelab2 using the gcloud CLI.
10. Connected to the second VM using gcloud compute ssh.

## Tools/Services Used

* Google Compute Engine
* Cloud Shell
* gcloud CLI
* NGINX Web Server
* SSH

## Results

Successfully created and managed virtual machine instances using both the Cloud Console and command-line interface. Installed and verified a running NGINX web server accessible through the VM's external IP address.
