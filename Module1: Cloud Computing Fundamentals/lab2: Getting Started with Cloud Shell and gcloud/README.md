# Getting Started with Cloud Shell and gcloud

## Objective

To learn how to use Cloud Shell and the gcloud command-line tool to manage Google Cloud resources, create virtual machines, configure firewall rules, and view system logs.

## Key Steps Performed

1. Activated Cloud Shell.
2. Configured default region and zone.
3. Retrieved project information using gcloud commands.
4. Created environment variables for project ID and zone.
5. Created a VM instance named gcelab2.
6. Connected to the VM using SSH.
7. Installed the Nginx web server.
8. Added VM network tags.
9. Created a firewall rule to allow HTTP traffic.
10. Verified web server accessibility.
11. Viewed and filtered Cloud Logging entries.

## Tools/Services Used

* Google Cloud Shell
* gcloud CLI
* Compute Engine
* Cloud Logging
* Firewall Rules

## Results

Successfully created and managed a Compute Engine VM using gcloud commands. Configured firewall access and verified that the Nginx web server was accessible through the VM's external IP address.
